Overview

Goal

The goal is to create a simple HTTP server called cihttp. HTTP is an application layer protocol on top of TCP/IP, so we need not worry about transporting the packets, handling fragmentation, switching packets, etc. We will see how data is encapsulated at this layer using the service/peer interfaces provided by HTTP.

We will not fully implement HTTP. We will implement three request types: GET, POST, and HEAD.

Milestones

  1. Implement the HTTP parser.
  2. Implement HTTP HEAD request/response.
  3. Implement HTTP GET request/response.
  4. Implement HTTP POST request/response.

Handling Requests/Generating Responses

HEAD

Only returns the header as if a normal GET request, BUT absolutely no body should be attached.

GET

  • Perform a HTTP/1.1 GET (see the RFC for details)
  • All file requests should search the www folder for the files.
  • A status code 404 should be sent when a file is not found. Along with some HTML to say "File not found"
  • Successful status code is 200 and the reason should be "OK"
  • Required headers are:
    • Content-length
    • Server (should say cihttp)
    • Last-modified
      • The last modified date in the format specified in the RFC (must be GMT!)

POST

  • Perform a HTTP/1.1 POST (see RFC for details)
  • To test a POST request, use form.html (after you have implemented GET)
  • The URL form.html submits to is post_test.
  • Store the keys and values from the POST request inside the post_data NVL structure of the request.
  • Reply with a simple response that contains the data in HTML.
